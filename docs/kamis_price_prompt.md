# 역할 및 목표
당신은 대한민국 농수산물 유통정보(KAMIS) API와 기상청 기후 데이터를 융합하여 시각화하는 전문가 수준의 프론트엔드 및 데이터 시각화 개발자입니다. 제공된 명세, 기후 예측 시나리오, 디자인 가이드를 준수하여 완벽한 단일 HTML 대시보드 코드를 생성하세요.

# 대시보드 제목 
- 기후 변화에 따른 농산물 가격 변화 분석 및 예측 서비스

# 데이터 분석 개요
- 2021년부터 2026년까지의 농산물 가격 데이터를 기반으로 기후 변화가 농산물 가격에 미치는 영향을 분석
- 기후 데이터를 기반으로 농산물 가격 변화를 예측
- 기후 변화 시나리오를 통해 농산물 가격 변화를 예측 

# 가설 설정
- 평균기온이 계절 평균보다 높을수록 채소류의 평균 가격은 상승한다.
- 강수량이 평년보다 많을수록 농산물 가격 변동성이 증가한다.
- 폭염일수가 증가할수록 채소류 가격은 상승한다.
- 폭우 발생 후 반입물량은 감소하고 농산물 가격은 상승한다.
- 폭설 발생 후 노지 채소의 가격은 단기간 상승한다.
- 기후 변화가 가격에 미치는 영향은 품목별로 다르다.

# 데이터의 범위 및 기간
- **데이터 범위:** KAMIS에서 제공하는 **농산물 전체 품목(부류코드 100: 식량작물, 200: 채소류, 400: 과일류)**의 데이터를 모두 커버할 수 있도록 파이프라인을 구성할 것
- **데이터 기간:** API 데이터를 가져올 때 시작일과 종료일을 입력받을 것 (2021.01.01 ~ 2025.12.31)

# 기술 스택
- HTML5, Tailwind CSS (CDN 연동, forms 및 container-queries 플러그인 포함)
- Google Fonts: Space Grotesk (디지털 숫자/헤드라인용), Inter (본문용)
- Icons: Google Material Symbols Outlined (발광 효과 매핑)

* **테마 컬러 팔레트 익스텐션:**
  - `primary`: #006d32, `primary-container`: #00d166, `on-primary`: #ffffff
  - `secondary`: #0059bb, `secondary-container`: #0070ea

# 🌡️ 기후 변화 예측 및 기간 선택 시뮬레이터 위젯 구성
대시보드 상단 또는 메인 컨트롤 패널 옆에 사용자가 직관적인 시나리오와 미래 타겟 기간을 마우스 클릭만으로 손쉽게 조합하여 미래 농산물 가격을 추론할 수 있는 **'안티그래비티 스타일 시뮬레이터 카드'**를 추가해줘.

1. **사용자 입력 인터페이스 (UI 컴포넌트):**
   - **[단계 1] 기후 시나리오 선택 (럭셔리한 라디오 버튼 또는 토글 시스템):**
     * ○ 평년 (Normal)
     * ○ 폭염 (Heatwave)
     * ○ 집중호우 (Heavy Rain)
     * ○ 가뭄 (Drought)
     * ○ 폭설 (Heavy Snow)
   - **[단계 2] 예측 대상 기간 선택 (드롭다운 또는 세그먼트 버튼 컴포넌트):**
     * 2026년 하반기 (2026.07 ~ 2026.08)
     * 예측 기간은 주 단위로 진행


2. **시뮬레이션 데이터 추론 인터페이스 (JS 로직 - `updateClimateSimulation`):**
   - 사용자가 시나리오 라디오 버튼이나 기간 선택 요소를 변경하면 기후 민감도 가중치와 기간별 계절성(Seasonal) 충격량이 결합되어 자동으로 미래 가격을 계산하도록 설계해줘.
   - 예: 가뭄 시나리오에서 '2026년 하반기(김장철 포함)' 선택 시 채소류(배추, 양배추 등)의 가격 변동 충격량이 더욱 가파르게 누적 가중되는 비선형 로직을 시뮬레이션할 것.
   - 변동된 추론 데이터는 하단의 품목별 예측 가격 및 미래 예측 점선 그래프에 즉시 실시간 동적 연동될 것.

# 카드 내부 UI 요구사항
안티그래비티 테마 안에서 각 카드의 내부 요소는 레이아웃의 시인성을 위해 아래 구조를 명확히 유지해줘. (왼쪽에서 오른쪽 순서)
* **[왼쪽 구역] 품목 및 단위:** 품목명은 고딕 계열 굵은 폰트, 단위는 작은 회색 폰트.
* **[중앙 구역] 주간 가격 및 기후 예측 데이터:**
    - **상단 (과거 시세):** '지난주 평균가' 태그와 해당 가격 배치.
    - **하단 (미래 예측):** '선택 기간 예측' 태그와 기후 변수 및 기간 가중치가 반영되어 실시간으로 변하는 계산된 미래 예상 가격 배치. (우측 정렬)
* **[오른쪽 구역] 변동률 및 인디케이터:** 가격 등락을 나타내는 네온 화살표 인디케이터 조건부 분기 처리 스타일 설계.

# 가격 및 기후 추이 그래프 고도화
기존의 최근 7일간의 가격 흐름을 보여주는 부드러운 네온 라인 차트(인라인 SVG) 자리를 확장하여, **'선택한 미래 예측 기간의 기후 예측 점선 그래프'**가 부드럽게 이어지도록 그리도록 설계해줘.
- 과거 7일선은 실선으로 표현하고, 시뮬레이터 조건(시나리오 + 기간)에 따라 출렁이는 미래 예측 선은 점선 효과(`stroke-dasharray="4 4"`)를 주어 예측선임을 명확히 시각화할 것.
- 그래프 배경에 예측 오차 범위를 나타내는 반투명한 그라데이션 신뢰구간(Confidence Interval) 영역 그라데이션 필터 적용.

# 데이터 구조 및 예측 모델 가중치 (전체 농산물 확장형)
전체 농산물(과일류, 채소류, 식량작물) API 연동 실패 시나리오 및 초기 화면 렌더링을 위해 자바스크립트 내부에 부류별 대표 품목군이 포함된 확장형 Mock JSON 데이터를 선언해줘.
1. **식량작물 (부류코드 100):** 감자, 고구마 등 (기온/강수 민감도: 보통)
2. **채소류 (부류코드 200):** 양배추, 배추, 상추 등 (기온/강수 민감도: 매우 높음, 폭염/폭우 시 가격 40% 이상 폭등 서칭 알고리즘 반영)
3. **과일류 (부류코드 400):** 사과, 배, 체리/아보카도 수입 등 (가뭄/폭염 민감도: 높음, 생육기 기후 인자 반영)

# 보안 및 API 자격증명 처리 규칙 (필수)
- 모든 API 자격증명(KAMIS, 가락시장, 기상청 ASOS) 정보는 소스코드 내부 및 문서 공유 공간에 노출하지 않아야 합니다.
- 프로젝트 루트 경로의 `.env` 파일에 각 키를 정의하고 백엔드 서버(`server.js`)가 프록시하는 아키텍처를 적용하세요.
  * `KAMIS_ID`: 발급받은 KAMIS 회원 ID (예: `YOUR_KAMIS_ID`)
  * `KAMIS_KEY`: 발급받은 인증 Cert Key (예: `YOUR_KAMIS_CERT_KEY`)
  * `GARAK_ID`: 가락시장 API ID (예: `YOUR_GARAK_ID`)
  * `GARAK_KEY`: 가락시장 API KEY (예: `YOUR_GARAK_KEY`)
  * `WEATHER_API_KEY`: 기상청 ASOS 일자료 API KEY (예: `YOUR_WEATHER_API_KEY`)
- 프론트엔드 자바스크립트는 외부에 키를 절대 노출하지 않고, 로컬 프록시 경로(`/api/price`, `/api/volume`, `/api/weather`)와 연동합니다.
